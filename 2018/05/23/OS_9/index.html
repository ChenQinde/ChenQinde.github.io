虚拟内存

发表于 2018-05-23 | 分类于 操作系统

背景

  1. 虚拟内存定义：在具有层次结构存储器的计算机系统中，采用自动实现部分装入和部分对换功能，为用户提供一个比物理主存容量大得多的，可寻址的一种“主存储器”。

  2. 好处：

    1. 在主存中保留多个进程：由于对任何特定的进程都仅仅装入它的某些块，因此就有足够的空间放置更多进程。
    2. 进程可以比主存的全部空间还大：通过分段或分页的虚拟内存，程序员可以获得巨大的主存，大小与磁盘储存器相关。

按需调页 Demand Paging

  1. 定义：在需要的时候才调入相应的页进入主存

  2. 懒惰交换 lazy swapper 和 交换程序 swapper 的区别：懒惰交换只有当需要单个页的时候才交换，而交换程序是对整个进程的所有页进行交换操作

基本概念

  1. 有效-无效位：在页表中，如果相关页既合法又在内存中则标记为有效，否则标记为无效（有可能是不合法，也有可能是不在内存中）

  2. 页错误陷阱 page-fault trap：对标记为无效的访问会产生页错误陷阱，此时操作系统会检查进程的内部页表，以确定引用是否合法，如果是非法的引用的话，则终止进程，如果是合法的话，便找一个空闲帧，将磁盘中的东西放进那个帧里，还有修改页表和内部页表，最后重新开始因陷阱中断的进程（这一段过程中，我们的进程是在休眠，感受不到操作系统在做的事情，另外这个重新开始，好像真的就是重新开始，而没有保存的机制）。

按需调页的性能

  1. 有效访问时间 = (1-p)×ma+p×页错误时间

    ps. ma是内存访问时间

写时复制 Copy-On-Write

  1. 定义：Copy-on-Write (COW) allows both parent and child processes to initially share the same pages in memory. If either process modifies a shared page, only then is the page copied.(一开始共用一个，修改的时候再一人一个，互不冲突，相当于懒惰复制)

  2. 操作系统为写时复制请求提供了空闲缓冲池 pool，通常采用按需填零 zero-fill-on-demand去分配这些页，在分配之前先填零，清楚以前的内容。

页面置换 Page Replacement

基本页置换

  1. 定义：当用户进程需要用到某一个不在内存的页时，有空闲帧的时候就用空闲帧，否则就使用页置换算法选择一个victim frame，再将它写进磁盘中，改变页表和帧表，得到一个空帧，然后再从硬盘中将需要使用的页写入该空帧，最后重启用户进程。

  2. 上面那个方法的缺点在于需要两次页传输（一次换入、一次换出），可以通过使用修改位（脏位）降低开销，即只有当该页被修改后才需要换入，否则直接清空就好了。

FIFO页置换

  1. 定义：总是淘汰最先调入主存的那一页，或者说在主存中驻留时间最长的那一页（常驻的除外）。

  2. Belady异常：对有的页置换算法，页错误率可能会随着所分配的帧数的增加而增加，而原期望是增加内存能改善性能。

最优置换 Optimal Page-replacement Algorithm

  1. 定义：调入一页而必须淘汰一个旧页时，所淘汰的页应该是以后不再访问的页或距现在最长时间后再访问的页，能保证最低可能的页错误率。

  2. 难题：理论上不可以实现，因为无法预知未来。

  3. 应用：可以用来比较别的算法的优劣。

LRU页置换

  1. 定义：淘汰的页面是在最近一段时间里较久未被访问的那页。

  2. 难题：如何对页进行排序

  3. 解决方法：需要有一定的硬件支持

    1. 计数器：每个页表项关联一个使用时间域，每次对内存引用，所有没被使用的页的时间域都会加一，时间域最大的就是LRU页
    2. 栈：每次使用页表，就将该页表放到栈顶，此时栈底就是LRU页

ps. LRU页置换和最优置换都没有Belady异常，因为两者都属于栈算法，都不会有Belady异常

Counting Algorithm

  1. Keep a counter of the number of references that have been made to each page.

  2. 最不经常使用页置换算法 LFU Algorithm：replaces page with smallest count.（最经常使用的最有可能使用）

  3. 最常使用页置换算法 MFU Algorithm：based on the argument that the page with the smallest count was probably just brought in and has yet to be used.（最不经常使用的可能是还没开始使用）

帧分配

帧的最少数量

  1. 分析：必须分配至少最少数量的帧，首先必须有足够的帧来容纳所有单个指令所引用的页，否则会有指令无法进行，而单个指令所需的页数是由给定计算机结构定义的。

  2. 每个进程的最少数量是由体系结构决定的，而最大数量是由可用物理内存的数量决定的。

分配算法

  1. 平均分配 equal allocation

  2. 比例分配 proportional allocation（根据进程相对大小或者进程优先级）

全局分配与局部分配

  1. global replacement：Process selects a replacement frame from the set of all frames; one process can take a frame from another.

    1. 缺点：进程不能控制其页错误率，导致一个进程的位于内存的页集合不仅取决于进程本身的调页行为，还取决于其他进程的调页行为

  2. local replacement：Each process selects from only its own set of allocated frames.

    1. 不能很好利用其他进程中不常用的内存，导致系统吞吐量低

系统颠簸 Thrashing

  1. 定义：频繁的页调度行为导致一个进程在换页上用的时间要多于执行时间(a process is busy swapping pages in and out)

系统颠簸的原因

  1. 原因：操作系统在监视CPU的使用率，如果CPU使用率太小，那么向系统引入新的进程，采用全局置换算法，过多的页错误导致CPU使用率低，而CPU使用率低又会导致操作系统增加多道程序的程度，使情况更加差。

  2. 解决方法：局部置换算法

  3. 局部模型 locality model

    1. 定义：当进程执行时，它从一个局部移向另一个局部（局部是一个经常使用页的集合），一个程序通常由多个不同局部组成，有可能重叠。
    2. Why does thrashing occur? the sum of size of locality > total memory size

工作集合模型

  1. 工作集合模型是基于局部性假设的，使用参数Δ(a fixed number of page references)定义工作集合窗口(working-set window)，最近Δ个引用的页集合称为工作集合，如果一个页不再使用，则从该工作集合中删掉，因此工作集合是程序局部的近似。

  2. WSSi(working set of Process Pi) = total number of pages referenced in the most recent working set

  3. D = sum of WSSi

    1. if D > m, then thrashing and we should suspend one of the processes

页错误率 Page-Fault Frequency(PFF)

  1. Establish “acceptable” page-fault rate
    1. If actual rate too low, process loses frame
    2. If actual rate too high, process gains frame
