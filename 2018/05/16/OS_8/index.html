主存

发表于 2018-05-16 | 分类于 操作系统

背景

基本硬件

  1. Main memory and registers are only storage CPU can access directly.

  2. 因为机器指令只能用内存地址作为参数，而不能用磁盘地址，所以执行指令以及指令使用的数据必须在内存中才行。

  3. CPU寄存器通常在一个CPU时钟周期内可以完成访问，但是内存访问需要多个CPU时钟周期。

  4. Cache sits between main memory and CPU registers.

  5. Protection of memory required to ensure correct operation.

  6. 采用基地址寄存器 base register 和界限地址寄存器 limit register 进行地址保护 address protection，如果用户模式下试图访问操作系统内存，则会陷入到操作系统中，并作为寻址错误处理。

地址绑定 Binding of Instructions and Data to Memory

  1. 分类：
    1. 编译时 Compile time: If memory location known a priori, absolute code can be generated; must recompile code if starting location changes - 如果开始地址发生改变，需要重新编译代码才行
    2. 加载时 Load time: Must generate relocatable code if memory location is not known at compile time - 如果开始地址发生改变，只需重新加载便可
    3. 执行时 Execution time: Binding delayed until run time if the process can be moved during its execution from one memory segment to another. Need hardware support for address maps (e.g., base and limit registers) - 执行时可以移来移去

ps. 本章主要就是在讲如何实现这些地址绑定

逻辑地址空间和物理地址空间 Logical vs. Physical Address Space

  1. 逻辑地址：由CPU生成、使用的地址，也被称为 virtual address（我们平时看见和使用的是逻辑地址）

  2. 物理地址：内存单元物理实际地址（我们知道它存在，但是我们却看不见它）

  3. 逻辑地址空间：由程序所生成的所有逻辑地址的集合

  4. 物理地址空间：这些逻辑地址所映射（利用MMU）的物理地址的集合

  5. 内存管理单元 Memory-management unit(MMU)：将基地址寄存器作为重定位寄存器，用户进程所生成的地址在送交内存之前，都将加上重定位寄存器的值，所以用户程序不会见到物理地址，只会看到逻辑地址。

    1. 定义：Hardware device that maps virtual to physical address
    2. 详细：In MMU scheme, the value in the relocation register is added to every address generated by a user process at the time it is sent to memory
    3. The user program deals with logical addresses; it never sees the real physical addresses
    4. MMU maps logical address dynamically.

  6. Logical and physical addresses are the same in compile-time and load-time address-binding schemes; logical(virtual) and physical addresses differ in execution-time address-binding scheme.（？）

动态加载 Dynamic Loading

  1. 定义：除了将一整个程序都放在内存中才能运行的方式，还存在一种称为动态加载的方式，子程序只有在调用时才加载，所有的子程序都以重定位的形式保存在磁盘中（将一个程序拆成几个子程序）。

  2. 优点：Better memory-space utilization; unused routine is never loaded. Useful when large amounts of code are needed to handle infrequently occurring cases.

  3. 不需要操作系统的帮助

动态链接 Dynamic Linking

  1. 定义：将链接延迟到运行时，一般适用于系统库等动态链接库 dynamically linked library（简称dll）

  2. 对比静态链接 static linking：

    1. 定义：由链接器在链接时将库的内容加入到可执行程序中
    2. 缺点：生成的可执行程序较大，载入内存时间较长

  3. 动态链接和动态加载的区别：不是将加载延迟到运行时，而是将链接延迟到运行时

  4. 二进制镜像对每个库程序的引用都有一个stub（used to locate the appropriate memory-resident library routine）Stub replaces itself with the address of the routine, and executes the routine.

  5. 需要操作系统的帮助

交换 Swap

  1. 定义：进程需要在内存中以便执行，但是可以暂时从内存交换到备份存储(backing store)中，当需要重新执行时再调回内存中。 A process can be swapped temporarily out of memory to a backing store, and then brought back into memory for continued execution.

  2. 备份存储 Backing store：fast disk large enough to accommodate copies of all memory images for all users; must provide direct access to these memory images.

  3. 滚出 roll out ，滚入 roll in：swapping variant used for priority-based scheduling algorithms; lower-priority process is swapped out so higher-priority process can be loaded and executed

  4. Major part of swap time is transfer time; total transfer time is directly proportional to the amount of memory swapped.

  5. System maintains a ready queue of ready-to-run processes which have memory images on disk.

连续内存分配 Contiguous Memory Allocation

  1. 内存分为两个区域：操作系统和用户进程，操作系统通常位于低地址（和 interrupt vector 一起），用户进程通常位于高地址

  2. 定义：每个进程都位于一个连续的内存区域

内存分配

  1. 方法一：multiple-partition method，即将内存分为多个固定大小的分区，每个分区只能容纳一个进程，当有空位时，就选择一个进程将它放进去空位中（最简单的固定分区，分页是比较强的固定分区）

  2. 方法二：variable-partition，即操作系统有一张表用来记录哪些内存可用，哪些内存已被占用，当有新的进程需要内存时，为该进程寻找足够大的孔，当终止时释放内存（动态分区）

  3. 动态内存分配问题：

    1. 首次适应
    2. 最佳适应
    3. 最差适应

碎片

  1. 外部碎片问题：

    1. 产生：首次适应和最佳适应都会产生这个问题
    2. 定义：当可用内存之和满足请求，但并不连续的现象
    3. 解决方法：
      1. 只有当重定位是动态并在运行时可以采用紧缩（开销很大）
      2. 允许物理地址空间是非连续的

  2. 内部碎片问题：

    1. 产生：都会产生
    2. 定义：通常将内存以固定大小的块为单位去分配，这样的话，进程所分配的内存可能比需要的更大

分页

  1. 核心方法：允许进程的物理地址空间是非连续的

  2. 简介：大小不等的固定分区和大小可变的分区效率低下，会产生内部或外部碎片。若将主存划分为固定大小块，并且块较小，每个进程也被分为同样大小的小块，可以将进程的块（称为页）指定到内存中的可用块（称为帧或页帧），可能形成内部碎片（在最后一页），没有外部碎片。（切成比较小的固定分区，一个进程可以分布在不连续的物理内存中）

基本方法

  1. 将物理内存分为固定大小的块，称为帧（Frame）- Divide physical memory into fixed-sized blocks called frames(size is power of 2, between 512 bytes and 8,192 bytes).

  2. 将逻辑内存也分为同样大小的块，称为页（Page）

  3. 页对应帧需要页表（存放在内存中），其中包含每个页所在物理内存的基地址

  4. 由CPU生成的每个地址分为：页号（p）和页偏移（d），页号直接等于页表中的索引

  5. 如果一个逻辑地址高m-n位作为页号，低n位作为页偏移，那么这个逻辑地址可以指向的空间总大小为$2^m$，页大小为$2^n$

  6. 帧表：保存物理内存（帧）的分配情况

  7. 采用分页类似于使用一组基（重定位）寄存器，每个基地址对应着一个内存帧。

硬件支持

  1. 绝大多数操作系统都为每个进程分配一个页表

  2. 页表硬件实现方式：

    1. 最简单的方法是放进专用寄存器中（当页表比较小的时候）
    2. 将页表放进内存中
      1. 页表基寄存器（PTBR）- 利用它指向页表，但是这种情况需要两次内存访问，一次根据寄存器中的值找到页表的地址，一次利用页表中写的数据地址访问内存字节
      2. 页表长度寄存器（PTLR） - 将它用于表示页表的大小，该寄存器的值用于检查每个逻辑地址以验证其是否位于进程的有效范围内
    3. 转换表缓冲区（TLB）的条目由键和值组成，页表放进内存中，TLB包含页表一部分的内容
      1. 放在小但是专用且快速的硬件缓冲中
      2. 使用：当CPU产生逻辑地址后，先找TLB，找不到的话（TLB失效），再找页表
      3. 地址空间标识符 ASID：用来为进程提供地址空间保护，防止一个进程访问另外一个进程的页表
      4. 缺点：
        1. 在TLB中找的时候，虽然快，但是需要一个个找（而不是像页表一样，直接用页表号就能找到），直到找不到的时候，才会找页表，这样的话反而会更加慢，所以现在应用硬件支持，支持每个项一个电路，一次过匹配全部的项，所以那张图中，TLB每个项都有一个箭头

保护

  1. 在页表中添加别的位来保护页，如：
    1. 使用一个位（保护位）来标明页可读写性
    2. 使用有效-无效位来标明该页是否在进程的逻辑地址空间中

共享页

  1. 使用分页技术的好处之一：可允许多个进程共享公共代码

  2. 缺点：

    1. 很难将进程切得这么完美（很难用语义去切）
    2. 如果对于某一块来说，上面一块和下面一块都被共享出去，那么这一块就不能变化，不能变大，也不能变小

页表结构

层次页表 Hierachical Paging

  1. 遇到的问题：整个页表过于大，无法连续的放进内存中，这样无法直接使用页号转变成偏移直接访问页表，得到对应的帧号

  2. 两级分页算法：外部页表、页表都是在内存里，外部页表是连续的，整个页表不是连续的，分成一部分一部分连续的，外部页表每个项指着一部分连续的页表（即将页表也进行分页）

  3. 比如32位地址中，22位用来做页号（前12位为外部页表，后10位为页表），剩余10位为页偏移

哈希页表 Hashed Page Tables

  1. 遇到的问题：超过32位地址用层次页表也不好，一般用哈希页表

  2. 哈希页表每个页号所指向的都是一条链表，如果要增加同页号的页时，则在链表后面加，链表每个元素有三个域：虚拟页号、所映射的帧号和指向链表中下一个元素的指针

反向页表 Inverted Page Tables

  1. 遇到的问题：每个进程有一个页表（对应每个物理空间），所以需要很多页表，这些表需要消耗很多内存空间，但是这些表指的都只是一个物理内存空间而已

  2. 反向页表对每个真正使用的内存页或帧才有一个项，整个系统只有一个反向页表（优点：节省存储空间）

  3. 每个逻辑地址包含三个域<process-id, page-number, offset>，每个反向页表包含两个域<process-id, page-number>，然后利用逻辑地址去找到对应的反向页表项（缺点：增加搜索时间），再根据那个项的位置，得到物理地址的帧号

  4. 还有一个缺点是内存共享比较麻烦
