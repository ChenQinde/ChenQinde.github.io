用于大型程序的工具

发表于 2018-05-24 | 分类于 C++

背景

  1. 异常处理用来分离问题的检查与解决过程
  2. 命名空间用来防止名字冲突提供

异常处理

目的：将问题的检查与解决过程分离开，程序一部分专门负责问题的检测，然后将解决问题的任务传递给另一部分

抛出异常

  1. 简介：C++中使用throw语句抛出异常，程序的执行权转移到与之匹配的catch语句中（可能在同一函数中，也可能在外面）

  2. 栈展开过程：如果throw在try语句块中，那么会寻找对应的catch语句中，如果匹配不到，则退出该函数，在调用当前函数的外层函数继续寻找，直到主函数，如果一直没有匹配到的话，则调用标准库函数terminate来终止程序进行

  3. 栈展开过程中局部对象（块里面的对象）被自动销毁，但是如果是动态分配的对象，记得要自己释放内存，否则会出现内存泄漏的情况

  4. 异常对象：throw对异常对象进行拷贝初始化，故throw语句中的表达式必须拥有完全类型，异常对象处于编译器管理的空间中（和别的对象不一样），当被处理完后，异常对象会被销毁

捕获异常

  1. 性质：

    1. 查找第一个匹配的处理代码，所以派生类异常的处理代码应该放在基类异常的处理代码前面，越是专门的处理放在越前的位置

    2. 如果在该catch语句块中无法处理完异常，则使用throw;语句重新抛出该异常便可

      ps. 如果在catch语句外使用空的throw;语句，编译器会调用terminate

    3. 使用catch(...)可以捕捉到所有的异常

异常类层次

  1. 继承体系第二层将exception分成两个大类：运行时错误和逻辑错误，其中运行时错误表示在程序运行时才能检测到的错误，逻辑错误指我们可以在程序代码中发现的错误

  2. 自定义异常类型可以继承于标准库异常类exception，还可以写一个普通的类（不用说像Java一样，一定要继承于Exception类才行）

命名空间定义

  1. 简介：包含关键词namespace，随后是命名空间的名字，在命名空间名字后面是一系列声明和定义（类、变量、函数、模板和其他命名空间）

  2. 注意：

    1. 命名空间既可以定义在全局作用域中，也可以定义在其他命名空间中，但是不能定义在函数或者类的内部
    2. 命名空间作用域后不需要分号
    3. 不能将include某个头文件的语句放进自己定义的命名空间中，如果这样做了，会把头文件所有名字定义成该命名空间中的成员

  3. 命名空间可以是不连续的，这样的话，可以一个命名空间放在头文件中，用来放置声明，另外一个放在源文件中，用来放置定义，然后别的文件想用的话，只需要include头文件便可（接口与实现分离）

  4. 全局命名空间：隐式声明，使用::member_name去引用，member_name代表全局命名空间中的一个成员

使用命名空间成员

  1. 最简单的方法是使用::去一层层的使用

  2. 命名空间的别名：使用 namespace test = haha; 格式去起别名；注意！命名空间可以有多个别名，所有别名都等价！

  3. using声明有效范围从声明的地方开始，到using声明所在的作用域结束为止，都可以直接用声明的名字

  4. using指示可以直接使用整个命名空间内的所有成员
