Inheritance 继承

Example

公有、私有和受保护继承

分析

  1. 对基类成员的访问权限只与基类的访问说明符有关，和派生访问说明符没有关系。

  2. 派生访问说明符的作用在于控制派生类用户（包括派生类的派生类在内）对于基类成员的访问权限。

    • public：派生类用户可以按照基类的访问说明符去访问基类的成员

    • protected：派生类用户无法访问任何基类的成员，但是派生类的成员和友元可以访问

    • private：派生类用户无法访问任何基类的成员

重点

  1. 基类将自己的方法分成两类：一类是希望派生类保持不变，另外一类是希望派生类能改写成适合自己的版本，后者被声明为virtual。

  2. virtual 需要在基类中就被实现（和Java中的 abstract 不一样），除非是纯虚函数。

  3. 任何除构造函数外的非 static 方法，都可以被声明为 virtual。

  4. 作为继承关系中根节点的类通常都会定义一个虚析构函数。

  5. 派生类只能访问基类的 public 和 protected 的成员。

  6. protected 和 private 成员类似，对于类的用户来说是不可访问的，和 public 成员类似，对于派生类的成员和友元来说是可以访问的，但是也是只能通过派生类对象来访问基类的受保护成员。

  7. 每个类控制自己成员初始化的过程，所以派生类不会管理基类成员初始化。

  8. 友元关系无法继承

Polymorphism 多态

Dynamic Binding

在运行的时候，会对 virtual 函数进行动态绑定

Class Definition

  1. 派生类的声明中包含类名但是不包含它的派生类列表。
  1. 想用某个类为基类时，必须先实现这个类再使用。

类型转换

derived-to-base ：将派生类对象绑定到基类的引用或者指针上（隐式）

base-to-derived ：通过static_cast（强制）和dynamic_cast（安全）将基类的引用或者指针转换为派生类（显式）

Note :

  1. 转换只对引用或者指针生效

  2. derived-to-base 转换过程中会发生 sliced down 的现象，即派生类部分会被忽略，只有基类部分会被拷贝、移动或赋值

  3. 只有当派生类是公有继承基类时，用户才可以使用derived-to-base，除此之外不能使用该转换

  4. 无论使用什么方式继承，派生类的成员函数和友元都可以使用derived-to-base

  5. 只有当派生类是公有或保护继承积累时，派生类的派生类的成员函数和友元才能使用derive-to-base

虚函数

  1. 虚函数一定要被定义，因为只有在运行时才知道调用了哪个版本的虚函数。

  2. 覆盖虚函数时必须形参列表和返回类型都相匹配（返回类型可以从基类的指针或者引用换成派生类的，但要求派生类到基类的类型转换是可访问的）

  3. final 和 override 都是放在形参列表和尾置返回类型之后

  4. 基类和派生类中虚函数的默认参数最好保持一致，否则由本次调用的静态类型决定，而并非是动态类型

  5. 使用作用域运算符可以强行使用某个版本的虚函数（小心没有用作用域运算符，然后调用的仍是自己的虚函数，导致无限递归的发生）

抽象基类

含有纯虚函数的类是抽象基类，相当于是接口，不能直接创建抽象基类的对象，但其实仍有创建，只是在派生类的对象生成中隐式创建
