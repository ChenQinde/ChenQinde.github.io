简介

它的简单定义：

GraphQL 是一种用于 API 的查询语言，和传统的前后端交互方式不同的是，它可以使得前端能够准确地获得它需要的数据，没有任何冗余。

它具有的特点：

  • 请求所要的数据不多不少

  • 获取多个资源只用一个请求

  • 描述所有可能的类型系统

例子

在服务计算课上，我们小组进行了一次简单 web 服务与客户端开发实战：

https://github.com/DalaoBringMeFly/BringMeFlyServer

  • 服务端选择使用 GraphQL 来构建服务。

  • 客户端选择使用 Vue 富客户端 js 框架来构建我们的前端。

我负责服务端的 GraphQL 与 boltDB 的交互处理。
