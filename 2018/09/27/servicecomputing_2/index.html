前言

以前我一直以为 Git 和 Github 是同一样东西，后来第一次和别人合作写代码，需要使用 Git 来管理代码后才懂得原来 Git 是一个分布式版本控制系统，而 Github 是一个代码仓库（开源协作社区），利用 Git 可以来远程使用他，这一篇文章是一篇总结性的笔记（总结了各路很厉害的博客），用来以后给自己看（好吧，主要是用来交作业啦）。

工作空间

  1. 新建工作空间

ps. Linux中$HOME是一个环境变量，代表当前登录的用户的主文件夹，使用echo $HOME可以知道具体代表的哪个目录

  1. 工作空间的解释（摘抄自参考资料1）：

工作空间其实就是一个目录，其中包含三个子目录：

  • src 目录包含Go的源文件，它们被组织成包（每个目录都对应一个包）
  • pkg 目录包含包对象
  • bin 目录包含可执行命令

go 工具用于构建源码包，并将其生成的二进制文件安装到 pkg 和 bin 目录中。

src 子目录通常包会含多种版本控制的代码仓库（例如Git或Mercurial）， 以此来跟踪一个或多个源码包的开发。
