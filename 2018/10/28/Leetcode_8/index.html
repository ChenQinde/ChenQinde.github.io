135 Candy

评价

困难

题目

There are N children standing in a line. Each child is assigned a rating value.

You are giving candies to these children subjected to the following requirements:

Each child must have at least one candy.

Children with a higher rating get more candies than their neighbors.

What is the minimum candies you must give?

题目理解

第一次做评分是 Hard 的题目，开始的时候非常害怕，但是看了题目后，感觉好像也不是非常的难，题目的意思是 N 个孩子排一行，给每个孩子评一个分数，然后再把糖给每个孩子，其中每个孩子至少有一颗糖，如果两个相邻的孩子分数不同，分数高的必须分更加多的糖，最后要求最少需要派糖的数量。

版本 1

用比较暴力的方法，首先初始化数组，每个孩子都有一颗糖果，接着一次次的遍历数组，改正不合规则的地方（分数高的孩子糖果数少），直至每个孩子手上的数目合规则时退出遍历，但是这种方法负责度太过高，对于那种从左到右由高到低的数组，复杂度是 O(n^2)，因为需要遍历数组 n 遍才行。

结果：Time limit Exceeded

版本 2

接着我就放弃挣扎了，开始看答案的代码，他维护了两个数组，其中一组保证对于每个孩子，左边的孩子与他之间是符合规则的，另一个数组保证了对于每个孩子，右边的孩子与他之间是符合规则的，接着给予每个孩子的糖果数从两个数组中取最大值便可，具体的证明如下所示：

To prove this is correct, it basically comes down to showing that in the result array (taking max of the right-to-left result and left-to-right result) preserves both invariants. Here to show one case:
Denote the result arrays from left-to-right pass as LR[n], right-to-left RL[n].
Suppose rating[i] < rating[i + 1]. Then in the left-to-right pass, the result LR[i] < LR[i + 1]. In the right-to-left pass, since it’s non-increasing from element i + 1 to i, RL[i] = 1 <= RL[i + 1]. So max(LR[i], RL[i]) = LR[i] < LR[i + 1] <= max(LR[i + 1), RL[i + 1]). That is result[i] < result[i + 1].

一些小记录

c++ 中有 max 函数，可以用来比较两个对象的大小，但是需要 #include<algorithm> 才行。
