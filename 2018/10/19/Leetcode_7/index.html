153 Find Minimum in Rotated Sorted Array

评价

中等

题目

Suppose an array sorted in ascending order is rotated at some pivot unknown to you beforehand.

(i.e., [0,1,2,4,5,6,7] might become [4,5,6,7,0,1,2]).

Find the minimum element.

You may assume no duplicate exists in the array.

题目理解

给出一个已经排好顺序的数组，然后将这个数组进行平移，平移的距离未知，我们需要找到数组中最小的一位。

但是后来我突然想到一个理论上更加好的方法，复杂度是 O(logn)，我们可以用二分法进行处理，我们设置三个pivot来表示数组的三个位置：start、mid、end，当 nums[start] 小于或者等于 nums[mid] 时，证明在 start 和 mid 之间是顺序的，所以最小的应该 mid + 1 和 end 之间，故继续搜索 mid + 1 和 end 之间。

但是这么写很容易错的一点（一开始我也错了）是，需要处理只剩下两个元素的边界情况：如果第一个大于第二个，我们需要搜索第二个，如果第一个小于第二个，则继续搜索第一个，要实现这个目标，需要设置 nums[start] 小于或者等于 nums[mid]，而并非小于，这里需要注意。
