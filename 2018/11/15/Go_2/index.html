mux解析

什么是 mux？

github 中的 readme 文件解释 mux 是一个实现了 URL 的路由和调度的包，其中 mux 这个名字意思是 multiplexer，即多路复用器，就像计算机网络中，出发时将多个端口输出的内容都并在一路，到达终点时，再将他们分开到目标端口。

主要特性

  • 实现了 http.Handler 接口，因此它兼容于 http.ServeMux；

  • 不仅支持正则路由还可以按照Method，header，host等信息匹配；

  • Registered URLs can be built, or “reversed”, which helps maintaining references to resources.

  • Routes can be used as subrouters: nested routes are only tested if the parent route matches. This is useful to define groups of routes that share common conditions like a host, a path prefix or other repeated attributes. As a bonus, this optimizes request matching.

源码文件结构

前言

mux 中并非所有代码都会进行解析，只会解析我觉得重要代码（其实是我看得懂的代码），省略的代码用 … 表示，详细代码可以在 github 进行阅读。

mux.go

Router 结构体：将路由信息存放在一个 Route 对象指针的数组中，数组中的每一个 Route 对象都表示一条路由信息。

  • NotFoundHandler：当没有 Route 匹配时的操作

  • routes：存放包含的所有 Route

  • namedRoutes：所有存放着的 Route 的名字

Match 函数：当请求 req 来到时，会遍历 routes 数组，调用 Route 的 Match 函数，找到第一个匹配的 Route 时则返回 True。

NewRoute 函数：用来给 Router 加入新的 Route

route.go

Route 结构体：Route 对象

  • handler 是对应这个 Route 的 http.Handler

  • matchers 包含了验证与这个 Route 的匹配需要的所有方法

Match 函数：根据 Route 中包含的所有 matcher 来判断是否和请求 req 匹配，如果有一项不匹配则返回 False，全部匹配则返回 True，并将信息都放进 match 中来返回。
