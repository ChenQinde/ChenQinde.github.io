树结构索引

发表于 2018-05-14 | 分类于 数据库

1 出现原因

单层索引到多层索引的递进

2 索引顺序存取方法 ISAM

  1. 结构特点：静态

  2. 数据存储：ISAM索引的数据项存放在树的叶子页 leaf page 和串链到叶子页的其他溢出页中

2.1 数据更改

  1. 新建：Leaf (data) pages allocated sequentially, sorted by search key then index pages then overflow pages.

  2. 查询：Start at root and use key comparisons to go to leaf.

  3. 插入：Find leaf that data entry belongs to, and put it there. Overflow page if necessary.

  4. 删除：Seek and destroy! If deleting a tuple empties an overflow page, de-allocate it and remove from linked-list.

  5. 注意点：

    1. 静态树结构中的插入和删除不改变索引页的数据项和主叶子页的数量，这样的话在并发访问时，不用对索引页加锁，所以相比起B+树，当数据分布和大小稳定且溢出链很少时，ISAM的效果更好
    2. 插入数据时，每个叶子页里面的都是不用管顺序关系，有空位就放进去
    3. 因为页的数目在创建树时是已知的，并且在随后的插入和删除也不改变，所以不需要指向下一叶子页的指针

3 B+树

问题：静态结构会随着文件的增长产生很长的溢出链，从而导致性能变差

3.1 结构特点

  1. 秩 order：每个节点中的数据项都必须在 $[\frac{order}{2},order]$之间，所以可以保证最小50%的占有率

  2. 平衡树：Each path from the root to any leaf has the same length

3.2 功能

  1. 查询：

    1. Equality Search
    2. Range Search：先找到第一个满足的项，接着利用sibling pointers去找到满足整个范围的项

  2. 插入：

    1. 选择子树
    2. 若有位置则插入就好
    3. 如果节点满了，就需要分裂，分裂成两部分，大小分别为 d 和 d + 1
    4. 中间码被推上去，然后继续分裂，如果一直往上，使root被分裂则会使树高度加一
    5. 注意：叶子页的分裂会生成新的索引项，而索引页的分裂不会生成新的索引项

  3. 删除：

    1. 查询
    2. 删除

    3. 如果不满足二分之一，则需要重分布，向别人借一个，删掉原索引项，借谁的就将谁的新的第一个作为索引项，一直往上，使root删掉则会使树高度减一（情况总共分为：叶子页的重分布，索引页的重分布，叶子页的合并，索引页的合并）

      ps.有个疑问：为什么索引页重分布的时候不是按借别人一个，而是按n和n+1去分

3.3 块加载B+树

  1. 前提：当数据记录集合已经存在，将要在某个字段上创建B+树索引

进程

发表于 2018-05-12 | 分类于 操作系统

进程概念

进程

  1. 非正式的定义：在运行中的程序（程序是死的，进程是活的）

  2. 正式的定义：进程包含程序代码（文字段），程序正在运行中的活动（程序计数器和寄存器的值），栈（用来存储临时数据），数据段（用来存储全局变量）和堆（用来动态分配内存）

进程状态

  1. New 新建：The process is being created

  2. Running 运行：Instructions are being executed

  3. Waiting 等待：The process is waiting for some event to occur (such as an I/O completion or reception of a signal)

  4. Ready 就绪：The process is waiting to be assigned to a processor

  5. Terminated 终止：The process has finished execution

Process Control Block 进程控制块

  1. 每个进程在操作系统中都是以process control block（PCB）的形式存在

Process Schduling 进程调度

必要性：分时系统用来在进程之间快速切换CPU，以便用户能与多个进程进行交互，这就需要进程调度来选择进程在CPU中运行

调度队列

分类：

  1. Job queue – set of all processes in the system

  2. Ready queue – set of all processes residing in main memory, ready and waiting to execute

  3. Device queues – set of processes waiting for an I/O device

Schedulers 调度程序

作用：用来将进程在不同调度队列间迁移

分类：

  1. Long-term scheduler (or job scheduler) 长期调度程序：selects processes from the pool and loads them into memory for execution

  2. Short-term scheduler (or CPU scheduler) 短期调度程序：selects from among the processes that are ready to execute and allocates the CPU to one of them

区别：

  1. 长期调度程序执行得不频繁（因为系统中新进程的创建不频繁）

  2. 短期调度程序执行得很频繁且速度很快

进程分为

生产者和消费者问题

  1. 简介：生产者进程产生信息以供消费者进程消费

  2. 例子：

    1. 编译器产生的汇编代码，汇编器将汇编代码产生目标代码
    2. 服务器产生html文件和图像，请求资源的客户Web浏览器进行消费

  3. 为了使这两个进程可以并行执行，采用共享内存，而共享内存又分为两种：

    1. 无限缓存：无限大小，消费者可能需要等待新的项，生产者可以无限生产新的项
    2. 有限缓存：缓冲大小一定，如果缓冲为0，消费者要等，如果缓冲满了，生产者需要等待

  4. 如果使用flag的话，肯定会空出一位出来

  5. 分析：

    1. 当 (in + 1) % BUFFER_SIZE == out 的时候，就满了
    2. 当 in == out 的时候，就为空
    3. 当加入一个缓冲时，in += 1
    4. 当减少一个缓冲时，out -= 1

in 指的位置是空的，比如假设 out 是0，当 in 是4时，判断该缓冲区满，但实际上 in 指的位置是空的，所以缓冲区实际上只有4个，如果我们再加一个，会导致 in 变为0，此时被判断成缓冲区是空的。

  1. 改进：设置一个 count 去记录非空位的大小

Multual exclusion: only one process at a time can use a resource.

Hold and wait: a process holding at least one resource is waiting to acquire additional resources held by other processes.

No preemption: a resource can be released only voluntarily by the process holding it, after that process has completed its task.

4. 循环等待（互相等待对方占有的资源，导致成环的现象出现）
