死锁

发表于 2018-05-09 | 分类于 操作系统

1 前言

1.1 目标

  1. To develop a description of deadlocks, which prevent sets of concurrent processes from completing their tasks.

  2. To present a number of different methods for preventing or avoiding deadlocks in a computer system.

1.2 初步了解

死锁发生必要条件 
1. 互斥

Multual exclusion: only one process at a time can use a resource.

2. 占有且等待（占有某资源的同时在等待别的资源）

Hold and wait: a process holding at least one resource is waiting to acquire additional resources held by other processes.

3. 非抢占（资源只能在进程完成任务后自动释放，而不能被其他进程抢占）

No preemption: a resource can be released only voluntarily by the process holding it, after that process has completed its task.

4. 循环等待（互相等待对方占有的资源，导致成环的现象出现）

ps.必须四个同时满足才会发生死锁

2 系统模型

2.1 系统描述

  • 资源类型 $R_1, R_2, \cdots, R_m$

    • 内存空间
    • CPU周期
    • 文件
    • I/O 设备

  • 每个资源类型 $R_i$ 有 $W_i$ 个实例

  • 使用资源的正常顺序

    • request 申请（如果申请不被立即允许，则会陷入等待）
    • use 使用
    • release 释放

2.2 资源分配图 Resource-Allocation Graph

定义：顶点 Vertex 和 边 Edge 的有向图

详细：

1. 顶点分为两类 进程（圆形） 和 资源（矩形），边也分为两类 申请边 和 分配边
2. 资源（矩形）中每个圆点代表一个实例

注意：

  1. 环中每一类资源如果只有一个实例时，会发生死锁。
  2. 如果在图中，有进程在使用完资源后可以释放该资源，解开环的话，就不是死锁状态，死锁状态是不能解开的。

3 死锁处理

3.1 分类

  1. Ensure that the system will never enter a deadlock state.

  2. Allow the system to enter a deadlock state and then recover.

  3. Ignore the problem and pretend that deadlocks never occur in the system; used by most operating systems, including UNIX.

4 死锁预防

从必要条件去解决死锁问题

  1. Mutual Exclusion

  2. Hold and Wait

  3. No preemption

  4. Circular wait

    • protected：派生类用户无法访问任何基类的成员，但是派生类的成员和友元可以访问

    • private：派生类用户无法访问任何基类的成员
