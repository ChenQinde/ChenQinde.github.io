主存

发表于 2018-05-16 | 分类于 操作系统

1 背景

1.5 动态链接 Dynamic Linking

  1. 定义：将链接延迟到运行时，一般适用于系统库等动态链接库 dynamically linked library

  2. 静态链接 static linking：

    1. 定义：由链接器在链接时将库的内容加入到可执行程序中
    2. 缺点：生成的可执行程序较大，载入内存时间较长

2 交换

  1. 基址寄存器 base register 和 限址寄存器 limit register 定义内存的逻辑地址空间

  2. 两种地址（操作系统中的MMU将两者相互映射）：

    1. 逻辑地址：由CPU生成、使用的地址，也被称为 virtual address（我们平时看见和使用的是逻辑地址）
    2. 物理地址：内存单元物理实际地址（我们知道它存在，但是我们却看不见它）

3 连续内存分配

  1. 内存分为两个区域：操作系统和用户进程，操作系统通常位于低地址（和 interrupt vector 一起），用户进程通常位于高地址

  2. 定义：每个进程位于一个连续的内存区域

3.1 内存映射与保护

  1. Memory Management Unit (MMU)
    1. 定义：Hardware device that maps virtual to physical address
    2. 详细：In MMU scheme, the value in the relocation register is added to every address generated by a user process at the time it is sent to memory
    3. The user program deals with logical addresses; it never sees the real physical addresses

树结构索引

发表于 2018-05-14 | 分类于 数据库

1 出现原因

单层索引到多层索引的递进

2 索引顺序存取方法 ISAM

  1. 结构特点：静态

  2. 数据存储：ISAM索引的数据项存放在树的叶子页 leaf page 和串链到叶子页的其他溢出页中

2.1 数据更改

  1. 新建：Leaf (data) pages allocated sequentially, sorted by search key then index pages then overflow pages.

  2. 查询：Start at root and use key comparisons to go to leaf.

  3. 插入：Find leaf that data entry belongs to, and put it there. Overflow page if necessary.

  4. 删除：Seek and destroy! If deleting a tuple empties an overflow page, de-allocate it and remove from linked-list.

  5. 注意点：

    1. 静态树结构中的插入和删除不改变索引页的数据项和主叶子页的数量，这样的话在并发访问时，不用对索引页加锁，所以相比起B+树，当数据分布和大小稳定且溢出链很少时，ISAM的效果更好
    2. 插入数据时，每个叶子页里面的都是不用管顺序关系，有空位就放进去
    3. 因为页的数目在创建树时是已知的，并且在随后的插入和删除也不改变，所以不需要指向下一叶子页的指针

3 B+树

问题：静态结构会随着文件的增长产生很长的溢出链，从而导致性能变差

3.1 结构特点

  1. 秩 order：每个节点中的数据项都必须在 $[\frac{order}{2},order]$之间，所以可以保证最小50%的占有率

  2. 平衡树：Each path from the root to any leaf has the same length

3.2 功能

  1. 查询：

    1. Equality Search
    2. Range Search：先找到第一个满足的项，接着利用sibling pointers去找到满足整个范围的项

  2. 插入：

    1. 选择子树
    2. 若有位置则插入就好
    3. 如果节点满了，就需要分裂，分裂成两部分，大小分别为 d 和 d + 1
    4. 中间码被推上去，然后继续分裂，如果一直往上，使root被分裂则会使树高度加一
    5. 注意：叶子页的分裂会生成新的索引项，而索引页的分裂不会生成新的索引项

  3. 删除：

    1. 查询
    2. 删除

    3. 如果不满足二分之一，则需要重分布，向别人借一个，删掉原索引项，借谁的就将谁的新的第一个作为索引项，一直往上，使root删掉则会使树高度减一（情况总共分为：叶子页的重分布，索引页的重分布，叶子页的合并，索引页的合并）

      ps.有个疑问：为什么索引页重分布的时候不是按借别人一个，而是按n和n+1去分

3.3 块加载B+树

  1. 前提：当数据记录集合已经存在，将要在某个字段上创建B+树索引

发表于 2018-05-12 | 分类于 操作系统

3.4.1 生产者和消费者问题

  1. 简介：生产者进程产生信息以供消费者进程消费

  2. 例子：

    1. 编译器产生的汇编代码，汇编器将汇编代码产生目标代码
    2. 服务器产生html文件和图像，请求资源的客户Web浏览器进行消费

  3. 为了使这两个进程可以并行执行，采用共享内存，而共享内存又分为两种：

    1. 无限缓存：无限大小，消费者可能需要等待新的项，生产者可以无限生产新的项
    2. 有限缓存：缓冲大小一定，如果缓冲为0，消费者要等，如果缓冲满了，生产者需要等待

  4. 如果使用flag的话，肯定会空出一位出来

  5. 分析：

    1. 当 (in + 1) % BUFFER_SIZE == out 的时候，就满了
    2. 当 in == out 的时候，就为空
    3. 当加入一个缓冲时，in += 1
    4. 当减少一个缓冲时，out -= 1
  1. 改进：设置一个 count 去记录非空位的大小

1.1 目标

  1. To develop a description of deadlocks, which prevent sets of concurrent processes from completing their tasks.

  2. To present a number of different methods for preventing or avoiding deadlocks in a computer system.

1.2 初步了解

死锁发生必要条件 
1. 互斥

Multual exclusion: only one process at a time can use a resource.

2. 占有且等待（占有某资源的同时在等待别的资源）

Hold and wait: a process holding at least one resource is waiting to acquire additional resources held by other processes.

3. 非抢占（资源只能在进程完成任务后自动释放，而不能被其他进程抢占）

No preemption: a resource can be released only voluntarily by the process holding it, after that process has completed its task.

4. 循环等待（互相等待对方占有的资源，导致成环的现象出现）

进程同步

发表于 2018-05-09 | 分类于 操作系统

1 背景

使用 count 去解决生产者消费者问题时，如果生产者进程和消费者进程并发执行，则有可能无法保持数据一致性。因为两个程序都有自己的局部寄存器，当要改变 count 时，他们会取出 count 到各自的寄存器中，然后修改，再存进内存中，这样会使数据和与其不同。

定义：以上现象定义为竞争条件 race condition，即多个进程并发访问和操作同一数据且执行结果与访问发生的特定顺序有关。

2 临界区问题

  1. 定义：设计一个以便进程协作的协议

    • 进入区 entry section：请求允许进入其临界区
    • 临界区 critical section：可能改变共同变量、更新一个表、写一个文件
    • 退出区 exit section：退出临界区要执行的代码
    • 剩余区 remainder section

  2. 要求：

    • 互斥 Mutual Exclusion：只有一个进程可以在临界区执行
    • 前进 Progress：没有进程在临界区时，应允许一个需要进入临界区的进程进入
    • 有限等待 Bounded Waiting：不允许需要进入临界区的进程被无限延迟

  3. 处理操作系统中的临界区问题：

    • 抢占内核：允许处于内核模式的进程被抢占
    • 非抢占内核：处于内核模式的进程会一直运行，直到退出内核模式、阻塞才会轮到别的进程。
